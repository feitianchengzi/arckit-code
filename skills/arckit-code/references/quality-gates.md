# 质量门

## 验证选择

| 改动类型 | 最低验证 |
| --- | --- |
| 小文案/样式 | 静态检查或截图/预览 |
| 局部逻辑 | 相关单元测试或定向运行 |
| 共享服务/状态 | 单元测试 + 调用方检查 |
| 数据迁移/缓存 | 旧数据样本、空数据、损坏数据路径 |
| 网络/API | 成功、业务错误、网络错误、取消或超时 |
| UI 交互 | 正常、边界、快速重复操作 |
| 性能 | 复现路径、指标或手测场景 |
| 发布配置 | 配置文件、权限文案、端到端清单 |

## 禁止通过的状态

- 代码无法构建且未说明原因。
- View/render 路径中新增重型计算或副作用。
- 异步任务无法取消且会在页面切换后继续写状态。
- 错误被吞掉，用户或日志无法区分关键失败类型。
- 新数据结构没有旧版本兼容策略。
- 线上日志包含 token、密钥、用户敏感内容或大对象。

## 回归口径

验证不是越多越好，而是覆盖本次风险。优先验证用户路径、共享边界、数据兼容和错误恢复。
