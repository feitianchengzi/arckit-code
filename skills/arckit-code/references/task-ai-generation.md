# AI Generation 任务规则

## 目标

把 AI 生成做成可约束、可解析、可取消、可恢复、可控成本的产品链路。

## 决策规则

- 定义生成目标、输入参数、上下文边界和不可接受输出。
- Prompt 集中 builder/service 管理，不在 View 或页面逻辑里临时拼接。
- 输出 schema 可解码、可校验、可回退。
- streaming/SSE 有部分状态、取消、超时和失败恢复。
- 错误区分网络、认证、超时、取消、解析失败、校验失败、空结果、权益不足和模型拒绝。
- 涉及成本/权益时，生成前后状态明确。

## 反模式

- 直接信任模型输出作为业务数据。
- prompt、解析、质量门散落在多个页面。
- 重复点击产生并发生成和重复扣费。
- stream 中断没有恢复或用户可理解状态。

## 验证

覆盖 prompt builder、schema decode、领域校验、取消、重试、空结果、失败映射和成本状态。
