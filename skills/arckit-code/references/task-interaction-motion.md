# Interaction Motion 任务规则

## 目标

把手势、动画和复杂交互实现为可预期、可取消、不卡顿的状态机。

## 决策规则

- 列出页面所有交互和触发区域。
- 先定义手势优先级、互斥关系、方向锁和边界。
- 临时交互状态局部化，业务状态在结束或确认时提交。
- 动画边界明确，不让多个隐式动画叠加。
- 系统级滚动、缩放、输入和选择体验优先成熟平台控件。

## 反模式

- 拖拽过程中刷新全局业务状态。
- 手势结束、取消、边界回弹路径不完整。
- 为了“纯声明式”复刻成熟系统物理交互。
- 动画过程中触发网络、解析或重型数据链路。

## 验证

手测快速连续操作、取消、边界、反向滑动、缩放态冲突和低性能设备路径。
