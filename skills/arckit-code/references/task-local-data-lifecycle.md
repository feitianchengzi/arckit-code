# Local Data Lifecycle 任务规则

## 目标

定义本地数据从创建、读取、更新、迁移、损坏降级、共享到清理的完整生命周期。

## 决策规则

- 先分类数据：领域数据、偏好、凭据、缓存、草稿、历史、临时文件、导入导出、扩展共享数据。
- 每类数据明确存储位置、生命周期、清理策略和兼容策略。
- schema 变化必须有迁移或读取兼容路径。
- 缓存 payload 和 refresh metadata 分开建模。
- 只有成功获得业务可用数据后才更新刷新时间。
- 离线写入、远端同步、后台刷新和冲突解决必须定义队列、幂等、重试和最终一致性边界。

## 反模式

- 用一个通用 `updatedAt` 判断所有 payload 新鲜度。
- 旧数据损坏直接阻断主路径。
- 空响应默认覆盖已有缓存。
- Widget/扩展读取 App 私有不可访问路径。
- 同步失败后丢失本地未提交变更。

## 验证

使用旧版本样本、空数据、损坏数据、缓存过期、并发刷新、离线写入、冲突合并和后台恢复场景验证。
