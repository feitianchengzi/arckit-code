# Media Pipeline 任务规则

## 目标

把普通图片、音频、视频和上传做成稳定管线，而不是只把资源显示出来。

## 决策规则

- 先区分缩略图、详情大图、头像、上传、扩展资源和播放器。
- 资源 identity、缓存 key、fallback、缩略图和原图关系必须稳定。
- 远程资源有 loading、success、failure、fallback success、retrying 状态。
- 上传前处理格式、方向、尺寸、体积和 mime type。
- 媒体状态和缓存策略服务列表、详情、头像和上传中的真实路径。

## 反模式

- 图片加载散落在页面中，没有统一失败和缓存策略。
- fallback 成功后阻止更高质量资源补全。
- 上传层接收平台 UI 图片类型作为通用接口。
- 把系统分享素材、二维码识别和系统级图片查看器混进普通远程图片组件。

## 验证

覆盖加载失败、fallback、重试、缓存命中、内存压力、上传压缩和文件不可访问。
