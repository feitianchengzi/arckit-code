# Navigation Routing 任务规则

## 目标

统一内部导航和外部入口，让路由可解析、可生成、可恢复、可测试。

## 决策规则

- 统一路由模型，外部入口都进入同一个 parser/router。
- 区分内部状态导航、外部 deeplink、push/widget/share 入口和 modal/sheet。
- 路由解析和生成需要 round-trip 规则。
- 冷启动、热启动、登录态缺失、目标不存在和权限不足有降级路径。
- 路由不应承载大量业务 payload，优先传稳定 id。

## 反模式

- 每个入口各自解析 URL。
- View 直接读取外部 URL 并跳转。
- 路由状态和页面局部状态互相复制。
- deeplink 只测热启动，不测冷启动和登录态。

## 验证

覆盖解析、生成、冷启动、热启动、目标缺失、权限不足和重复打开。
