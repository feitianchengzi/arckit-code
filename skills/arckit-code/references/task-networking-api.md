# Networking API 任务规则

## 目标

把远端 API 做成可取消、可重试、可映射错误、可测试的数据通道。该任务不包含 AI prompt 质量门，但可为 AI stream 提供底层 HTTP 能力。

## 决策规则

- API client 负责传输、鉴权、基础错误和日志。
- Feature service 负责业务接口组合和 DTO 到领域模型映射。
- View 不直接拼 URL、处理 token、解析底层错误码。
- 错误分类至少区分网络、认证、权限、业务错误、超时、取消和解析失败。
- 分页、上传、下载和重试必须定义幂等和取消边界。

## 反模式

- DTO 直接穿透到 UI 长期使用。
- 每个页面自己维护一套错误文案映射。
- refresh token 并发刷新没有 single-flight。
- 失败重试无上限或无用户可见状态。

## 验证

覆盖成功、业务错误、认证失败、弱网/超时、取消、空数据和解析失败。
