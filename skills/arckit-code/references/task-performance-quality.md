# Performance Quality 任务规则

## 目标

定位并治理卡顿、掉帧、首屏慢、列表慢和质量回归。该任务先找具体风险路径，再决定最小修复。

## 决策规则

- 确认卡顿路径：首屏、滚动、切页、手势、动画、图片、网络返回、数据解析。
- 渲染路径不做排序、过滤、解码、格式化器创建、文件 IO、同步图片处理。
- 复杂展示数据前置到 display model、缓存、service 或异步任务。
- 可观察状态粒度控制刷新范围。
- 异步任务有去重、取消和并发边界。

## 反模式

- 没有复现路径就大规模重构。
- 性能修复只靠“感觉更轻”。
- 列表行加载大图或每次重建重对象。
- 页面消失后任务继续写 UI 状态。

## 验证

使用复现步骤、指标、测试、手测或 Instruments 建议证明关键路径已改善或可观测。
