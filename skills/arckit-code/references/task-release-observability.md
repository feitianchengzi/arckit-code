# Release Observability 任务规则

## 目标

让发布相关代码和配置、隐私声明文件、日志、埋点、崩溃上下文和线上反馈形成可验证闭环。

## 决策规则

- 发布能力必须成套：代码、配置文件、后台、服务端文件和验证路径一致。
- 隐私声明和权限文案与真实使用匹配。
- 关键用户路径有开始、成功、失败、取消或等价观测。
- 日志分类清楚且脱敏。
- 线上问题需要能定位版本、入口、关键参数和错误分类。

## 反模式

- 只改本地代码，不识别对应外部后台配置。
- 日志写入 token、密钥、完整用户输入或大对象。
- 埋点事件名不稳定，属性无边界。
- 发布前没有冷启动、权限拒绝、弱网和外部入口清单。

## 验证

使用发布前清单、TestFlight/预发验证、日志样本、崩溃上下文和指标看板核对。
