# View Composition 任务规则

## 目标

把页面和组件拆成可维护结构，并覆盖真实 UI 状态。该任务关注页面职责、组件边界、设计系统、可访问性、多语言和状态矩阵。

## 决策规则

- 按视觉区域、业务职责、状态复杂度和复用需求拆分，不按控件数量机械拆。
- 子组件输入使用 display model、简单值、binding 和动作闭包。
- 页面协调数据和业务动作，子组件避免直接访问页面级 service、查询对象或全局状态。
- 稳定视觉值接入项目设计系统或 tokens。
- 长文本、空态、错误态、权限态、离线态和大字体必须有可理解表现。

## 反模式

- 一个页面文件同时包含多个独立页面。
- 子组件依赖父级完整状态对象。
- 视觉常量散落在页面中。
- 只验证 happy path，忽略空、错、长文本和辅助功能。

## 验证

优先使用 preview、截图、组件测试或手测矩阵验证关键状态。
